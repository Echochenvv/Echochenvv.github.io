从零搭建个人博客的一些思考

折腾了一圈，最终选了 Hexo + GitHub Pages 的方案。原因很简单：够轻、够快、够稳。

之前用过 WordPress，太重了。也试过 Next.js，对于一个纯文字博客来说有点杀鸡用牛刀。Hexo 的好处在于，你只需要关心 Markdown，剩下的交给模板和部署流程。

为什么不用现成的平台

掘金、知乎、Medium 都能写，但总觉得不够自由。自己的博客想放什么放什么，想怎么排版就怎么排版，不用考虑平台的推荐算法和审核机制。

技术选型

  • Hexo: 静态站点生成，写完 Markdown 直接构建
  • GitHub Pages: 免费托管，自动部署
  • 自定义主题: 不用现成主题，自己写 CSS，掌控每一像素

接下来的计划

慢慢写，不着急。先把框架搭好，内容后面慢慢补。
